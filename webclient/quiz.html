2. What are benefits of HTTP caching?
(a) Caching improves user-perceived latency by eliminating network communication with the origin server.
(b) Caching reduces bandwidth consumption by avoiding the transmission of unnecessary network packets.
(c) Reduced bandwidth consumption indirectly reduces latency for uncached interactions by reducing network congestion.
(d) Caching can reduce load on origin servers and on intermediate proxies, further improving latency.
(e) Cache response might be different from what would be returned by direct communication with the origin server.

3. What can happen if an IP implementation decrements the incoming TTL and then tests for equal to 0?
(a) Router should never receive a datagram with a TTL of 0.
(b) The datagram will be dropped by packet filter as it attempts to enter forwarding plane.
(c) Doing decrement and then test would set the TTL to 255 and let the datagram continue.

4. How does traceroute calculate the RTT?
(a) Traceroute stores 12 bytes of data in the data portion of the UDP datagram, part of which is the time the datagram was sent.
(b) ICMP returns only first 8 bytes of the IP datagram that was in error. This 8 byte UDP header.
(c) The time value stored by traceroute is not returned in the ICMP error message.
(d) Traceroute saves the time when it sends the packet, and when ICMP reply is received, it fetches the current time and subtracts the two value to get the RTT.
